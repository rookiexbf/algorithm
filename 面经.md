# 从请求次数优化到前端缓存到前端部署

### 个人总结和体会：当项目或者网页中的静态资源请求过多，可以从 http 缓存的角度入手，借助缓存有效减少请求数量（第一次访问网站除外），通过内容 hash 精确控制缓存。进一步的优化：静态资源都通过 cdn 加速，根据内容 hash 修改文件名（非覆盖式的发布），先全量发布静态资源然后在灰度发布页面。
